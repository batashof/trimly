# Trimly — план проекта (краткий контекст)

> Это входная точка контекста проекта. Добавь этот файл в Project knowledge своего Claude Project. Подробности по каждому разделу — в папке `docs/` (тоже добавь всю папку в knowledge).
>
> ## 1. Суть проекта
>
> Пет-проект для портфолио: приложение для записи клиентов на стрижку.
>
> - Монетизации нет, бюджет — 0. Это жёсткое ограничение, влияет на выбор всех сторонних сервисов.
> - - Есть знакомый барбер, потенциально готовый пользоваться готовым продуктом.
>   - - Два интерфейса:
>     -   1. **Публичная страница записи** — без авторизации, ссылка в Instagram барбера. Клиент выбирает барбера → услугу → дату/время → обязательно оставляет имя и телефон.
>         2.   2. **Админ-панель барбершопа** — с авторизацией. Управление барберами, услугами, расписанием и записями.
>              3. - Масштаб MVP: сразу с расчётом на нескольких барберов (один шоп, несколько мастеров).
>                 - - Клиентам отправляется уведомление о записи через Telegram-бота (см. `docs/notifications-telegram.md`). Барберу отдельные уведомления не нужны — он смотрит записи в панели.
>                  
>                   - ## 2. Роли
>                  
>                   - - **ADMIN** — единственная роль в MVP. Управляет барберами (как данными, не аккаунтами), услугами, расписанием, видит все записи.
>                     - - **Барберы** — не логинятся сами, это профили-данные, которыми управляет ADMIN. Модель специально спроектирована так, чтобы потом добавить роль `BARBER` (отдельный логин мастера, видит только свои записи) без переделки схемы.
>                       - - **Клиент** — не авторизуется, работает только с публичной страницей.
>                        
>                         - ## 3. Технологический стек (кратко, детали — `docs/architecture.md`)
>                        
>                         - - **Frontend**: Next.js 14 (App Router), TypeScript, TailwindCSS + shadcn/ui, TanStack Query, React Hook Form + Zod. Деплой — **Vercel**.
>                           - - **Backend**: NestJS, TypeScript, Prisma ORM, Passport-JWT. Деплой — **Render** (free web service, обычный long-running процесс, не serverless).
>                             - - **БД**: PostgreSQL на **Neon** (serverless Postgres, free tier).
>                               - - **Уведомления**: Telegram Bot API (grammY), webhook на Render-бэкенде. Бесплатно без ограничений на таком объёме.
>                                 - - **Монорепо**: Turborepo (`apps/web`, `apps/api`, `packages/shared`).
>                                   - - **CI**: GitHub Actions — lint + typecheck + test на каждый PR.
>                                    
>                                     - ## 4. Принятые решения (не пересматривать без причины)
>                                    
>                                     - Полный лог с датами и обоснованием — `docs/decisions-log.md`. Коротко:
>                                    
>                                     - | Тема | Решение |
> |---|---|
> | Backend хостинг | Render, не Vercel serverless |
> | Frontend хостинг | Vercel, не GitHub Pages |
> | Роли | Только ADMIN на MVP, Barber отвязан от User |
> | Телефон клиента | Обязательное поле |
> | Уведомления клиенту | Telegram-бот с deep-link (не SMS, не Viber, не email) |
> | Уведомления барберу | Не нужны в MVP, смотрит в панели |
> | Масштаб | Мультибарберность заложена с самого начала |
